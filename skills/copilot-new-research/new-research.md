# 生成研究
根據使用者提供之主題與問題，用第一性原理拆解，並上網搜尋相關資料。將相關資料整理好，並開始思考規劃與撰寫。撰寫過程中可以參考`source-repo/`的原始碼，因為這些文件非常要求正確性，所以一切內容都要去`source-repo/`核對，不能網路上人云亦云。最後輸出至`research/`，按照以下規則ㄡ

## File Structure
```
+ research
|-+ versions
  |-+ <version>
    |-+ <level>
      |-- <topic>
+ source-repo 
```

## 重視
- 正確性：所有結論都來自於原始碼 `source-repo/` 而不是靠猜的，所有特定名詞都要附上英文，且用詞必須極度一致且正確。
- 多樣性：可以上網搜尋大家的討論，這句這些坊間資料旁徵博引，但不得違背正確性
- 結構性：需要很有結構，除了粗細分明、分門別類，也要注意敘述之順序，與整體之目標。
- 易讀性：假設讀者對此主題一無所知，在介紹任何概念之前，若引用了相關概念，需先清楚解釋為何引用，並先介紹概念再往下講，切莫到處插入新的名詞。
- 視覺化：盡量使用圖表或表格展示概念，但圖表和表格必須極度正確，但果無法確定，就不要使用。

## 輸出
預設繁體中文輸出。
如 File Structure 所示，所有輸出都放在research裡面。
- version: source-repo當前的版本，他的README 裡面應該都會有。
- level: 這裡指的是抽象層級，基本上就是一步步從事實往上搭建到最抽象的整體概念或推導概念，每個層級所負責之描述各不相同。命名自由，但得先確定沒有同層級之資料夾，才能創新的資料夾。然後也不是1,2,3,4往上的命名方式，抽象層級不應該是線性關係，以層級該有的名稱作為資寮夾名稱即可。
- topic: 這個資料夾往下應是朝狀之資料夾或文件，每次的研究會專注在一個topic，最外層是README.md，然後與該topic相關的文件或是其他東西會放在一起，比如章節、索引、等等，這邊是最自由的狀態，但一定也要有好的結構。
