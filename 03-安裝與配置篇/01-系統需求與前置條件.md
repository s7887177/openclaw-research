## 1. 硬體需求

OpenClaw 本身是一個 Node.js 應用程式，其核心行程的資源消耗相對輕量。然而，當您啟用沙箱（Sandbox）功能、載入多個通道（Channels）、或同時運行本地模型時，硬體需求將顯著提升。以下根據不同的使用場景提供建議配置。

### 1.1 最低硬體需求（僅雲端模型 + 單通道）

此配置適用於僅使用雲端 LLM API（如 OpenAI、Anthropic）且僅連接單一通訊通道的場景。

| 項目 | 最低需求 | 說明 |
|------|----------|------|
| **CPU** | 1 核心（x86_64 或 ARM64） | 單核即可運行核心服務 |
| **RAM** | 1 GB | Node.js 行程約佔 200-400 MB，系統保留約 600 MB |
| **儲存空間** | 5 GB | 含 Node.js 執行環境、依賴套件、日誌空間 |
| **網路頻寬** | 1 Mbps 上行 | 用於 API 呼叫與 Webhook 接收 |

```bash
# 快速檢查系統資源
# 查看 CPU 核心數
nproc

# 查看可用記憶體（MB）
free -m | grep Mem | awk '{print $2}'

# 查看可用磁碟空間
df -h /
```

### 1.2 建議硬體配置（多通道 + 沙箱）

當您需要連接多個通訊平台（WhatsApp + Telegram + Discord 等）並啟用沙箱功能來執行程式碼或操作瀏覽器時，建議使用以下配置。

| 項目 | 建議配置 | 說明 |
|------|----------|------|
| **CPU** | 2-4 核心（x86_64 或 ARM64） | 沙箱的 Docker 容器需要額外計算資源 |
| **RAM** | 4 GB | 沙箱容器約額外佔用 1-2 GB |
| **儲存空間** | 20 GB | Docker 映像檔約佔用 5-8 GB |
| **網路頻寬** | 5 Mbps 上行 | 多通道同時收發訊息 |

```bash
# 確認 CPU 架構
uname -m
# 期望輸出: x86_64 或 aarch64

# 確認 CPU 詳細資訊
lscpu | grep -E "^(Architecture|CPU\(s\)|Model name)"

# 確認記憶體配置
cat /proc/meminfo | head -5
```

### 1.3 進階硬體配置（本地模型 + 全功能）

如果您計畫使用 Ollama 運行本地語言模型（如 Llama 3、Mistral 等），硬體需求將大幅提高，尤其是 RAM 和 GPU 方面。

| 項目 | 建議配置 | 說明 |
|------|----------|------|
| **CPU** | 8+ 核心 | 模型推理在無 GPU 時高度依賴 CPU |
| **RAM** | 16-32 GB | 7B 模型約需 8 GB RAM；13B 模型約需 16 GB |
| **GPU（選配）** | NVIDIA GPU 8 GB+ VRAM | 強烈建議使用 GPU 加速推理 |
| **儲存空間** | 50-100 GB SSD | 模型檔案通常為 4-30 GB 不等 |
| **網路頻寬** | 10 Mbps 上行 | 首次下載模型需要較大頻寬 |

```bash
# 檢查是否有 NVIDIA GPU
nvidia-smi 2>/dev/null && echo "NVIDIA GPU 已偵測" || echo "未偵測到 NVIDIA GPU"

# 檢查 GPU 記憶體
nvidia-smi --query-gpu=name,memory.total --format=csv,noheader 2>/dev/null

# 檢查磁碟 I/O 性能（簡易測試）
dd if=/dev/zero of=test_io bs=1M count=256 oflag=direct 2>&1 | tail -1
rm -f test_io
```

### 1.4 雲端伺服器建議

若您選擇在雲端部署 OpenClaw，以下是各大雲服務商的建議機型：

| 雲服務商 | 基礎方案 | 進階方案 | 本地模型方案 |
|----------|----------|----------|-------------|
| **AWS** | t3.small (2 vCPU, 2 GB) | t3.medium (2 vCPU, 4 GB) | g4dn.xlarge (4 vCPU, 16 GB, T4 GPU) |
| **GCP** | e2-small (2 vCPU, 2 GB) | e2-medium (2 vCPU, 4 GB) | n1-standard-4 + T4 GPU |
| **Azure** | B1ms (1 vCPU, 2 GB) | B2s (2 vCPU, 4 GB) | NC4as_T4_v3 (4 vCPU, 28 GB, T4 GPU) |
| **DigitalOcean** | Basic 2 GB | Basic 4 GB | GPU Droplet |
| **Linode** | Linode 2 GB | Linode 4 GB | GPU Instance |

> **💡 提示**：對於個人使用或開發測試，月費 5-10 美元的基礎方案通常已足夠。生產環境建議至少使用 4 GB RAM 的方案。

### 1.5 樹莓派與 ARM 裝置

OpenClaw 支援 ARM64 架構，因此可以在樹莓派 4B（4 GB 以上版本）或其他 ARM64 裝置上運行。但請注意以下限制：

- **樹莓派 4B（4 GB）**：可運行核心服務 + 單通道，不建議啟用沙箱
- **樹莓派 4B（8 GB）**：可運行核心服務 + 多通道 + 輕量沙箱
- **樹莓派 5（8 GB）**：效能改善明顯，可勝任大多數場景
- **不建議在 ARM 裝置上運行本地模型**，效能極差

```bash
# 確認樹莓派型號
cat /proc/device-tree/model 2>/dev/null

# 確認 ARM 架構版本
uname -m
# 期望: aarch64 (ARM64)
```

## 2. 作業系統支援

OpenClaw 基於 Node.js 構建，理論上可在任何支援 Node.js 22+ 的作業系統上運行。但為了最佳相容性與穩定性，我們建議以下作業系統。

### 2.1 Linux（推薦）

Linux 是 OpenClaw 的首選運行環境，特別是在生產環境部署時。以下發行版已經過完整測試：

| 發行版 | 版本 | 支援狀態 | 備註 |
|--------|------|----------|------|
| **Ubuntu** | 22.04 LTS, 24.04 LTS | ✅ 完整支援 | 推薦用於生產環境 |
| **Debian** | 11 (Bullseye), 12 (Bookworm) | ✅ 完整支援 | 穩定可靠 |
| **CentOS Stream** | 9 | ✅ 完整支援 | 企業環境首選 |
| **Rocky Linux** | 9 | ✅ 完整支援 | CentOS 替代方案 |
| **Fedora** | 39, 40 | ✅ 完整支援 | 開發者友善 |
| **Alpine Linux** | 3.18+ | ⚠️ 部分支援 | Docker 映像基礎，部分原生功能受限 |
| **Arch Linux** | Rolling | ⚠️ 社群支援 | 滾動更新可能造成不相容 |

```bash
# 確認 Linux 發行版
cat /etc/os-release | grep -E "^(NAME|VERSION)"

# 確認核心版本（建議 5.10+）
uname -r

# 確認 systemd 版本（用於服務管理）
systemctl --version | head -1
```

#### Ubuntu/Debian 前置安裝

```bash
# 更新系統套件
sudo apt update && sudo apt upgrade -y

# 安裝基礎工具
sudo apt install -y \
  curl \
  wget \
  git \
  build-essential \
  ca-certificates \
  gnupg \
  lsb-release \
  software-properties-common \
  unzip \
  jq
```

#### CentOS/Rocky Linux 前置安裝

```bash
# 更新系統套件
sudo dnf update -y

# 安裝基礎工具
sudo dnf install -y \
  curl \
  wget \
  git \
  gcc \
  gcc-c++ \
  make \
  ca-certificates \
  unzip \
  jq

# 啟用 EPEL 倉庫（如需額外套件）
sudo dnf install -y epel-release
```

### 2.2 macOS

macOS 是開發者常用的本地開發環境。OpenClaw 在 macOS 上可正常運行，但建議使用 Docker Desktop 進行部署。

| 版本 | 支援狀態 | 備註 |
|------|----------|------|
| macOS 13 (Ventura) | ✅ 完整支援 | Intel 與 Apple Silicon 均支援 |
| macOS 14 (Sonoma) | ✅ 完整支援 | 推薦版本 |
| macOS 15 (Sequoia) | ✅ 完整支援 | 最新版本 |

```bash
# 確認 macOS 版本
sw_vers

# 確認處理器架構
uname -m
# Intel: x86_64
# Apple Silicon: arm64

# 安裝 Homebrew（如尚未安裝）
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 透過 Homebrew 安裝基礎工具
brew install git node npm docker docker-compose jq
```

#### macOS 特別注意事項

1. **Docker Desktop 授權**：Docker Desktop 對大型企業需要付費授權。個人開發者與小型團隊可免費使用。
2. **檔案系統效能**：macOS 的 Docker 掛載卷效能較 Linux 慢，若遇到效能問題，可使用 `:cached` 或 `:delegated` 掛載選項。
3. **網路限制**：macOS 的 Docker Desktop 不支援 `host` 網路模式，需使用 port mapping。
4. **Apple Silicon 相容性**：大多數 Docker 映像已支援 ARM64，若遇到不相容映像，可使用 `--platform linux/amd64` 透過 Rosetta 2 模擬。

```bash
# macOS Docker Desktop 配置建議
# 在 Docker Desktop > Settings > Resources 中設定：
# - CPUs: 至少 2 核心
# - Memory: 至少 4 GB
# - Disk image size: 至少 30 GB

# 確認 Docker Desktop 狀態
docker info | grep -E "(Server Version|Operating System|Total Memory)"
```

### 2.3 Windows（透過 WSL2）

OpenClaw **不直接支援原生 Windows**。您必須透過 WSL2（Windows Subsystem for Linux 2）來運行 OpenClaw。

| Windows 版本 | WSL2 支援 | 備註 |
|-------------|----------|------|
| Windows 10 (1903+) | ✅ | 需開啟虛擬化功能 |
| Windows 11 | ✅ | 原生整合最佳 |
| Windows Server 2019+ | ⚠️ | 可行但非官方測試環境 |

#### WSL2 安裝步驟

```powershell
# 在 PowerShell（以系統管理員身分執行）中執行：

# 步驟 1：啟用 WSL
wsl --install

# 步驟 2：設定 WSL2 為預設版本
wsl --set-default-version 2

# 步驟 3：安裝 Ubuntu 24.04（推薦）
wsl --install -d Ubuntu-24.04

# 步驟 4：確認安裝結果
wsl --list --verbose
# 確認 VERSION 欄位為 2
```

#### WSL2 記憶體限制設定

WSL2 預設會佔用大量系統記憶體。建議建立 `.wslconfig` 檔案限制資源使用：

```ini
# 在 Windows 使用者目錄下建立 %UserProfile%\.wslconfig
# 例如: C:\Users\YourName\.wslconfig

[wsl2]
memory=4GB          # 限制 WSL2 最多使用 4 GB RAM
processors=4        # 限制使用 4 個 CPU 核心
swap=2GB           # 交換空間大小
localhostForwarding=true  # 啟用 localhost 轉發

[experimental]
autoMemoryReclaim=gradual  # 自動回收未使用記憶體
```

```powershell
# 套用設定（需重啟 WSL）
wsl --shutdown
wsl
```

#### WSL2 + Docker Desktop 整合

```powershell
# 方法 1：使用 Docker Desktop 的 WSL2 後端（推薦）
# 在 Docker Desktop > Settings > General 中勾選:
# ☑ Use the WSL 2 based engine

# 在 Docker Desktop > Settings > Resources > WSL Integration 中:
# 勾選要整合的 Linux 發行版

# 方法 2：在 WSL2 中直接安裝 Docker Engine（進階）
# 進入 WSL2 Ubuntu 後，按照 Linux 安裝步驟操作
```

#### WSL2 特別注意事項

1. **檔案系統**：請將 OpenClaw 的專案檔案放在 WSL2 的 Linux 檔案系統中（`/home/user/` 而非 `/mnt/c/`），否則 I/O 效能將極差。
2. **網路存取**：WSL2 使用獨立的虛擬網路。從 Windows 存取 WSL2 中的服務請使用 `localhost`。
3. **記憶體**：確保 WSL2 分配了足夠的記憶體（至少 2 GB）。
4. **時鐘漂移**：WSL2 可能出現時鐘漂移問題。如果遇到 API 請求失敗，嘗試同步時間：

```bash
# 在 WSL2 中同步系統時間
sudo hwclock -s
# 或
sudo ntpdate time.windows.com
```

## 3. 軟體依賴

### 3.1 Node.js 22+（建議 24+）

OpenClaw 需要 Node.js 22 或更高版本。我們強烈建議使用 Node.js 24（LTS），因為它包含了多項效能改進和安全修補。

#### 使用 nvm 安裝（推薦）

nvm（Node Version Manager）是管理多個 Node.js 版本的最佳工具：

```bash
# 安裝 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash

# 重新載入 shell 設定
source ~/.bashrc  # 或 source ~/.zshrc

# 確認 nvm 安裝成功
nvm --version

# 安裝 Node.js 24（推薦）
nvm install 24

# 設定預設版本
nvm alias default 24

# 確認版本
node --version
# 期望輸出: v24.x.x

npm --version
# 期望輸出: 10.x.x 或更高
```

#### 使用系統套件管理器安裝

##### Ubuntu/Debian

```bash
# 方法 1：使用 NodeSource 官方倉庫（推薦）
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt install -y nodejs

# 方法 2：使用 Snap
sudo snap install node --classic --channel=24

# 確認安裝
node --version
npm --version
```

##### CentOS/Rocky Linux

```bash
# 使用 NodeSource 官方倉庫
curl -fsSL https://rpm.nodesource.com/setup_24.x | sudo bash -
sudo dnf install -y nodejs

# 確認安裝
node --version
npm --version
```

##### macOS

```bash
# 使用 Homebrew
brew install node@24

# 確保 PATH 正確
echo 'export PATH="/opt/homebrew/opt/node@24/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc

# 確認安裝
node --version
npm --version
```

#### 使用 fnm 安裝（替代方案）

fnm（Fast Node Manager）是一個用 Rust 編寫的輕量快速 Node.js 版本管理器：

```bash
# 安裝 fnm
curl -fsSL https://fnm.vercel.app/install | bash

# 重新載入 shell
source ~/.bashrc

# 安裝 Node.js 24
fnm install 24
fnm default 24

# 確認
node --version
```

#### 版本相容性矩陣

| Node.js 版本 | OpenClaw 支援狀態 | 備註 |
|-------------|-------------------|------|
| 18.x | ❌ 不支援 | 缺少必要 API |
| 20.x | ❌ 不支援 | 缺少必要 API |
| 22.x | ✅ 支援（最低版本） | 功能完整但效能較差 |
| 23.x | ✅ 支援 | 非 LTS，不建議用於生產 |
| 24.x | ✅ 推薦 | 當前 LTS，最佳選擇 |

> **⚠️ 重要**：OpenClaw 使用了 Node.js 22 引入的多項新特性（如原生 `fetch`、`structuredClone` 等），因此無法在更低版本上運行。

### 3.2 npm 9+

npm 通常隨 Node.js 一起安裝。OpenClaw 需要 npm 9 或更高版本。

```bash

npm --version

# 如需升級 npm
npm install -g npm@latest

# 確認升級結果
```

```bash
# 恢復官方 registry
```

### 3.3 Docker 20.0+ 與 Docker Compose v2

Docker 是 OpenClaw 推薦的生產環境部署方式。您需要 Docker Engine 20.0+ 和 Docker Compose v2（作為 Docker CLI 插件）。

#### Docker 安裝（Linux）

```bash
# Ubuntu/Debian 安裝 Docker Engine
# 步驟 1：移除舊版本
sudo apt remove -y docker docker-engine docker.io containerd runc 2>/dev/null

# 步驟 2：安裝前置依賴
sudo apt update
sudo apt install -y \
  ca-certificates \
  curl \
  gnupg \
  lsb-release

# 步驟 3：添加 Docker 官方 GPG 金鑰
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 步驟 4：添加 Docker 倉庫
echo \
  "deb [arch="$(dpkg --print-architecture)" signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  "$(. /etc/os-release && echo "$VERSION_CODENAME")" stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 步驟 5：安裝 Docker Engine 和 Docker Compose
sudo apt update
sudo apt install -y \
  docker-ce \
  docker-ce-cli \
  containerd.io \
  docker-buildx-plugin \
  docker-compose-plugin

# 步驟 6：將當前使用者加入 docker 群組（免 sudo 執行 docker）
sudo usermod -aG docker $USER

# 重新登入或執行以下命令使群組生效
newgrp docker

# 步驟 7：驗證安裝
docker --version
# 期望輸出: Docker version 27.x.x
docker compose version
# 期望輸出: Docker Compose version v2.x.x

# 步驟 8：測試 Docker 是否正常運作
docker run --rm hello-world
```

#### Docker 安裝（CentOS/Rocky Linux）

```bash
# 移除舊版本
sudo dnf remove -y docker docker-client docker-client-latest \
  docker-common docker-latest docker-latest-logrotate \
  docker-logrotate docker-engine 2>/dev/null

# 安裝 dnf 工具
sudo dnf install -y dnf-plugins-core

# 添加 Docker 倉庫
sudo dnf config-manager --add-repo \
  https://download.docker.com/linux/centos/docker-ce.repo

# 安裝 Docker
sudo dnf install -y \
  docker-ce \
  docker-ce-cli \
  containerd.io \
  docker-buildx-plugin \
  docker-compose-plugin

# 啟動並設為開機自動啟動
sudo systemctl start docker
sudo systemctl enable docker

# 加入 docker 群組
sudo usermod -aG docker $USER
newgrp docker

# 驗證
docker --version
docker compose version
```

#### Docker 安裝後設定

```bash
# 設定 Docker daemon 配置
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  },
  "storage-driver": "overlay2",
  "default-address-pools": [
    {
      "base": "172.17.0.0/16",
      "size": 24
    }
  ]
}
EOF

# 重啟 Docker 使設定生效
sudo systemctl restart docker

# 驗證設定
docker info | grep -E "(Storage Driver|Logging Driver)"
```

#### Docker Compose v2 vs v1

> **⚠️ 注意**：OpenClaw 需要 Docker Compose **v2**（`docker compose` 命令，注意沒有 `-`），而非舊版的 `docker-compose`（有 `-`）。Docker Compose v2 作為 Docker CLI 的插件安裝，是隨 Docker Engine 一起安裝的。

```bash
# ✅ 正確：Docker Compose v2
docker compose version
docker compose up -d

# ❌ 錯誤：Docker Compose v1（已棄用）
docker-compose version   # 如果只有這個，需要升級
docker-compose up -d
```

### 3.4 Git 2.30+

Git 用於克隆 OpenClaw 倉庫和管理版本更新。

```bash
# 確認 Git 版本
git --version
# 期望輸出: git version 2.30+

# Ubuntu/Debian 安裝或升級
sudo apt install -y git

# CentOS/Rocky Linux
```

| `api.anthropic.com` | 443 | HTTPS | Anthropic API |
| `generativelanguage.googleapis.com` | 443 | HTTPS | Google AI API |

| `api.telegram.org` | 443 | HTTPS | Telegram Bot API |
| `graph.facebook.com` | 443 | HTTPS | WhatsApp Business API |
| `discord.com/api` | 443 | HTTPS | Discord Bot API |
| `slack.com/api` | 443 | HTTPS | Slack Bot API |

```bash
# 測試出站連線
echo "=== 測試外部服務連線 ==="

for host in api.openai.com api.anthropic.com generativelanguage.googleapis.com \
  registry.npmjs.org github.com api.telegram.org; do
  if curl -s --connect-timeout 5 -o /dev/null -w "%{http_code}" "https://$host" | grep -qE "^[23]"; then
    echo "✅ $host - 連線正常"
  else
    echo "❌ $host - 連線失敗"
```

### 5.2 入站連線需求（Webhook）

如果您需要接收通訊平台的 Webhook（如 Telegram、WhatsApp），您的伺服器需要：

1. **公開 IP 位址或域名**：Webhook 需要一個可從外部存取的端點
2. **HTTPS**：所有主流通訊平台均要求 Webhook 使用 HTTPS
3. **有效的 SSL 憑證**：不接受自簽憑證（部分平台可透過設定支援）

```
入站連線架構圖：

通訊平台 (Telegram/WhatsApp/...)
       │
       ▼ HTTPS (443)
 ┌─────────────┐
 │  反向代理    │ (Nginx/Caddy)
 │  SSL 終止    │
 └──────┬──────┘
        │ HTTP (3000)
        ▼
 ┌─────────────┐
 │  OpenClaw   │
 │  Gateway    │
 └─────────────┘
```

#### 使用 ngrok 進行開發測試

```bash
sudo firewall-cmd --permanent --add-service=https

sudo firewall-cmd --reload
```

#### iptables（進階使用者）

```bash

# 允許 loopback
sudo iptables -A INPUT -i lo -j ACCEPT

sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT

sudo netfilter-persistent save
```

### 6.2 SSH 安全加固

```bash
# 備份 SSH 配置
sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.backup

# 編輯 SSH 配置
sudo tee -a /etc/ssh/sshd_config.d/hardening.conf << 'EOF'
# 停用密碼登入（僅允許金鑰）
PasswordAuthentication no

# 停用 root SSH 登入
PermitRootLogin no

# 限制登入嘗試次數
MaxAuthTries 3

# 限制同時連線數
MaxSessions 5

# 自動斷開閒置連線（300 秒）
ClientAliveInterval 300
ClientAliveCountMax 2

# 僅允許特定使用者
AllowUsers your_username

# 使用 Protocol 2
Protocol 2
EOF

# 重啟 SSH 服務
sudo systemctl restart sshd

# 設定 SSH 金鑰認證
# 在本地機器上產生金鑰對（如果尚未有）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 將公鑰複製到伺服器
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@your-server
```

### 6.3 Fail2Ban 安裝

### 6.4 反向代理設定（Nginx）

使用 Nginx 作為反向代理可以提供 SSL 終止、請求限流和安全標頭。

```bash
# 安裝 Nginx
sudo apt install -y nginx  # Ubuntu/Debian
sudo dnf install -y nginx  # CentOS/Rocky Linux

# 建立 OpenClaw 反向代理配置
sudo tee /etc/nginx/sites-available/openclaw << 'NGINX'
server {
    listen 80;
    server_name openclaw.yourdomain.com;

    # 自動重導向至 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name openclaw.yourdomain.com;

    # SSL 憑證（使用 Let's Encrypt）
    ssl_certificate /etc/letsencrypt/live/openclaw.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/openclaw.yourdomain.com/privkey.pem;

    # SSL 安全設定
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;

    # 安全標頭
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # 請求大小限制
    client_max_body_size 50M;

    # 反向代理到 OpenClaw
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # Webhook 逾時設定
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }

    # 請求限流
    limit_req_zone $binary_remote_addr zone=openclaw:10m rate=30r/m;
    location /webhook/ {
        limit_req zone=openclaw burst=10 nodelay;
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
NGINX

# 啟用配置
sudo ln -sf /etc/nginx/sites-available/openclaw /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# 測試配置
sudo nginx -t

# 重啟 Nginx
sudo systemctl restart nginx
sudo systemctl enable nginx
```

## 8. 前置條件檢查清單

在開始安裝 OpenClaw 之前，請使用以下腳本進行全面的前置條件檢查。這個腳本會自動檢查所有必要的軟體和配置。

### 8.1 自動檢查腳本

```bash
#!/usr/bin/env bash
# OpenClaw 前置條件檢查腳本
# 用法: bash check-prerequisites.sh

set -euo pipefail

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[0;33m'
BLUE='\033[0;34m'
NC='\033[0m' # No Color

PASS=0
WARN=0
FAIL=0

check() {
  local name="$1"
  local status="$2"
  local message="$3"

  case "$status" in
    pass)
      echo -e "${GREEN}✅ PASS${NC}  $name — $message"
      ((PASS++))
      ;;
    warn)
      echo -e "${YELLOW}⚠️  WARN${NC}  $name — $message"
      ((WARN++))
      ;;
    fail)
      echo -e "${RED}❌ FAIL${NC}  $name — $message"
      ((FAIL++))
      ;;
  esac
}

echo -e "${BLUE}========================================${NC}"
echo -e "${BLUE}  OpenClaw 前置條件檢查${NC}"
echo -e "${BLUE}========================================${NC}"
echo ""

# --- 作業系統 ---
echo -e "${BLUE}【作業系統】${NC}"
OS_NAME=$(uname -s)
OS_ARCH=$(uname -m)
if [[ "$OS_NAME" == "Linux" ]]; then
  check "作業系統" "pass" "Linux ($OS_ARCH)"
elif [[ "$OS_NAME" == "Darwin" ]]; then
  check "作業系統" "pass" "macOS ($OS_ARCH)"
else
  check "作業系統" "fail" "不支援的作業系統: $OS_NAME"
fi

# --- CPU ---
echo ""
echo -e "${BLUE}【硬體資源】${NC}"
CPU_CORES=$(nproc 2>/dev/null || sysctl -n hw.ncpu 2>/dev/null || echo "0")
if [[ "$CPU_CORES" -ge 2 ]]; then
  check "CPU 核心數" "pass" "${CPU_CORES} 核心"
elif [[ "$CPU_CORES" -ge 1 ]]; then
  check "CPU 核心數" "warn" "${CPU_CORES} 核心（建議 2+）"
else
  check "CPU 核心數" "fail" "無法偵測"
fi

# --- RAM ---
if [[ "$OS_NAME" == "Linux" ]]; then
  TOTAL_RAM_MB=$(free -m | awk '/Mem:/ {print $2}')
elif [[ "$OS_NAME" == "Darwin" ]]; then
  TOTAL_RAM_MB=$(( $(sysctl -n hw.memsize) / 1024 / 1024 ))
fi

if [[ "${TOTAL_RAM_MB:-0}" -ge 4096 ]]; then
  check "記憶體" "pass" "${TOTAL_RAM_MB} MB"
elif [[ "${TOTAL_RAM_MB:-0}" -ge 1024 ]]; then
  check "記憶體" "warn" "${TOTAL_RAM_MB} MB（建議 4 GB+）"
else
  check "記憶體" "fail" "${TOTAL_RAM_MB:-未知} MB（至少需要 1 GB）"
fi

# --- 磁碟空間 ---
DISK_AVAIL_GB=$(df -BG / | awk 'NR==2 {gsub("G",""); print $4}')
if [[ "${DISK_AVAIL_GB:-0}" -ge 20 ]]; then
  check "可用磁碟空間" "pass" "${DISK_AVAIL_GB} GB"
elif [[ "${DISK_AVAIL_GB:-0}" -ge 5 ]]; then
  check "可用磁碟空間" "warn" "${DISK_AVAIL_GB} GB（建議 20 GB+）"
else
  check "可用磁碟空間" "fail" "${DISK_AVAIL_GB:-未知} GB（至少需要 5 GB）"
fi

# --- Node.js ---
echo ""
echo -e "${BLUE}【軟體依賴】${NC}"
if command -v node &>/dev/null; then
  NODE_VER=$(node --version | sed 's/v//')
  NODE_MAJOR=$(echo "$NODE_VER" | cut -d. -f1)
  if [[ "$NODE_MAJOR" -ge 24 ]]; then
    check "Node.js" "pass" "v${NODE_VER}（推薦版本）"
  elif [[ "$NODE_MAJOR" -ge 22 ]]; then
    check "Node.js" "warn" "v${NODE_VER}（可用，但建議升級到 24+）"
  else
    check "Node.js" "fail" "v${NODE_VER}（需要 22+）"
  fi
else
  check "Node.js" "fail" "未安裝"
fi

# --- npm ---
if command -v npm &>/dev/null; then
  NPM_VER=$(npm --version)
  NPM_MAJOR=$(echo "$NPM_VER" | cut -d. -f1)
  if [[ "$NPM_MAJOR" -ge 9 ]]; then
    check "npm" "pass" "v${NPM_VER}"
  else
    check "npm" "fail" "v${NPM_VER}（需要 9+）"
  fi
else
  check "npm" "fail" "未安裝"
fi

# --- Docker ---
if command -v docker &>/dev/null; then
  DOCKER_VER=$(docker --version | grep -oP '\d+\.\d+\.\d+' | head -1)
  check "Docker" "pass" "v${DOCKER_VER}"
else
  check "Docker" "warn" "未安裝（Docker 安裝為選配，但推薦用於生產環境）"
fi

# --- Docker Compose ---
if docker compose version &>/dev/null 2>&1; then
  COMPOSE_VER=$(docker compose version --short 2>/dev/null || docker compose version | grep -oP '\d+\.\d+\.\d+')
  check "Docker Compose" "pass" "v${COMPOSE_VER}"
else
  check "Docker Compose" "warn" "未安裝（Docker Compose 為選配）"
fi

# --- Git ---
if command -v git &>/dev/null; then
  GIT_VER=$(git --version | grep -oP '\d+\.\d+\.\d+')
  GIT_MAJOR=$(echo "$GIT_VER" | cut -d. -f1)
  GIT_MINOR=$(echo "$GIT_VER" | cut -d. -f2)
  if [[ "$GIT_MAJOR" -ge 3 ]] || [[ "$GIT_MAJOR" -ge 2 && "$GIT_MINOR" -ge 30 ]]; then
    check "Git" "pass" "v${GIT_VER}"
  else
    check "Git" "fail" "v${GIT_VER}（需要 2.30+）"
  fi
else
  check "Git" "fail" "未安裝"
fi

# --- curl ---
if command -v curl &>/dev/null; then
  check "curl" "pass" "$(curl --version | head -1 | awk '{print $2}')"
else
  check "curl" "warn" "未安裝（建議安裝）"
fi

# --- jq ---
if command -v jq &>/dev/null; then
  check "jq" "pass" "$(jq --version 2>/dev/null || echo '已安裝')"
else
  check "jq" "warn" "未安裝（建議安裝）"
fi

# --- 網路連線 ---
echo ""
echo -e "${BLUE}【網路連線】${NC}"
for host in api.openai.com api.anthropic.com registry.npmjs.org github.com; do
  if curl -s --connect-timeout 5 -o /dev/null "https://$host" 2>/dev/null; then
    check "$host" "pass" "可連線"
  else
    check "$host" "fail" "無法連線"
  fi
done

# --- Ollama（選用）---
echo ""
echo -e "${BLUE}【選用元件】${NC}"
if command -v ollama &>/dev/null; then
  check "Ollama" "pass" "$(ollama --version 2>/dev/null || echo '已安裝')"
else
  check "Ollama" "warn" "未安裝（僅在使用本地模型時需要）"
fi

# --- 摘要 ---
echo ""
echo -e "${BLUE}========================================${NC}"
echo -e "${BLUE}  檢查摘要${NC}"
echo -e "${BLUE}========================================${NC}"
echo -e "  ${GREEN}通過${NC}: $PASS"
echo -e "  ${YELLOW}警告${NC}: $WARN"
echo -e "  ${RED}失敗${NC}: $FAIL"
echo ""

if [[ "$FAIL" -eq 0 ]]; then
  echo -e "${GREEN}🎉 所有必要條件已滿足！您可以開始安裝 OpenClaw。${NC}"
else
  echo -e "${RED}⚠️ 有 $FAIL 項必要條件未滿足，請先解決後再繼續安裝。${NC}"
fi
```
