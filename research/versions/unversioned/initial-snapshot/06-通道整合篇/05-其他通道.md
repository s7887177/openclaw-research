## 1. 概述

除了 Discord、WhatsApp 和 Telegram 三大主要通道外，OpenClaw 還支援多種其他通訊平台的整合。每個通道都有其獨特的優勢和使用場景。本文將介紹這些通道的配置方式，以及如何開發自定義通道。

支援的其他通道一覽：

| 通道 | 整合方式 | 穩定性 | 適合場景 |
|------|----------|--------|----------|
| Slack | 官方 Bolt SDK | 高 | 團隊協作 |
| iMessage | BlueBubbles/AppleScript | 中 | Apple 生態圈 |
| Signal | signal-cli | 中 | 隱私優先 |
| CLI | 內建 | 高 | 開發者/終端控 |
| Matrix | matrix-js-sdk | 中 | 去中心化/自架 |

## 2. Slack 整合

### 2.1 Slack Bot 建立

Slack 提供了完善的 Bot 框架（Bolt SDK），整合過程較為標準化：

```
Step 1: 前往 api.slack.com/apps
Step 2: 建立新應用 → From Scratch
Step 3: 填寫應用名稱和選擇 Workspace
Step 4: 在 OAuth & Permissions 中新增 Bot Token Scopes:
  - chat:write
  - channels:history
  - groups:history
  - im:history
  - im:write
  - users:read
Step 5: 安裝應用到 Workspace
Step 6: 複製 Bot User OAuth Token (xoxb-...)
Step 7: 在 Socket Mode 頁面啟用 Socket Mode
Step 8: 生成 App-Level Token (xapp-...)
```

### 2.2 OpenClaw Slack 配置

```json
{
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "${SLACK_BOT_TOKEN}",
      "appToken": "${SLACK_APP_TOKEN}",
      "signingSecret": "${SLACK_SIGNING_SECRET}",
      
      "connectionMode": "socket",
      
      "allowFrom": {
        "users": ["slack:U0123456789"],
        "channels": ["slack:C0123456789"]
      },
      
      "dmPolicy": {
        "enabled": true,
        "allowUnknownUsers": false
      },
      
      "channelPolicy": {
        "enabled": true,
        "triggerMode": "mention",
        "threadReply": true
      },
      
      "formatting": {
        "useBlocks": true,
        "codeBlockStyle": "slack",
        "emojiEnabled": true
      }
    }
  }
}
```

### 2.3 Slack 特色功能

Slack 有許多獨特的功能可以與 OpenClaw 結合：

**Thread 回覆**：OpenClaw 在 Slack 頻道中會以 Thread 方式回覆，保持頻道整潔：

```json
{
  "channelPolicy": {
    "threadReply": true,
    "broadcastReply": false
  }
}
```

**Slash Commands**：可以設定自定義斜線命令：

```
/claw ask [問題] - 直接問 OpenClaw 問題
/claw summarize  - 摘要當前頻道的最近訊息
/claw translate  - 翻譯選取的訊息
```

**Block Kit 格式化**：Slack 的 Block Kit 提供了豐富的訊息排版：

```json
{
  "blocks": [
    {
      "type": "section",
      "text": {
        "type": "mrkdwn",
        "text": "🐾 *EasonClaw 回覆*\n\n以下是搜尋結果..."
      }
    },
    {
      "type": "divider"
    },
    {
      "type": "actions",
      "elements": [
        {
          "type": "button",
          "text": { "type": "plain_text", "text": "👍 有幫助" },
          "action_id": "feedback_helpful"
        }
      ]
    }
  ]
}
```

## 3. iMessage 整合

### 3.1 iMessage 整合方式

iMessage 沒有官方的 Bot API，整合方式依賴第三方工具：

| 方式 | 平台 | 說明 |
|------|------|------|
| BlueBubbles | macOS Server | 透過 macOS Messages.app 轉發 |
| AppleScript | macOS | 直接腳本控制 Messages.app |
| Beeper | 跨平台 | 通過 Beeper 橋接（需付費） |

### 3.2 BlueBubbles 設定

BlueBubbles 是最可靠的 iMessage 整合方案，需要一台始終運行的 Mac 電腦：

```
Step 1: 在 Mac 上安裝 BlueBubbles Server
  → brew install --cask bluebubbles
  
Step 2: 配置 BlueBubbles
  → 設定端口和認證
  → 連接 iCloud 帳號
  
Step 3: 確認 Messages.app 已登入 iMessage

Step 4: 取得 BlueBubbles API URL 和密碼
  → 通常是 http://localhost:1234
  → 記錄 API 密碼
```

### 3.3 OpenClaw iMessage 配置

```json
{
  "channels": {
    "imessage": {
      "enabled": true,
      "provider": "bluebubbles",
      "apiUrl": "http://localhost:1234",
      "password": "${BLUEBUBBLES_PASSWORD}",
      
      "allowFrom": {
        "users": [
          "imessage:+886912345678",
          "imessage:user@icloud.com"
        ]
      },
      
      "dmPolicy": {
        "enabled": true,
        "allowUnknownUsers": false
      },
      
      "features": {
        "tapback": true,
        "richLinks": false,
        "readReceipts": true
      }
    }
  }
}
```

### 3.4 限制與注意事項

- **需要 Mac 電腦**：BlueBubbles 需要一台始終運行的 Mac
- **Apple 帳號風險**：不正當使用可能導致 Apple 帳號被鎖
- **穩定性**：依賴 Messages.app 的穩定性
- **不支援群組**：Bot 功能在群組中受限
- **無法發送多媒體**：文字訊息是最可靠的

## 4. Signal 整合

### 4.1 Signal 的隱私優勢

Signal 是隱私保護最強的通訊平台：

- 端到端加密（開源協議）
- 不收集使用者資料
- 訊息不存儲在伺服器
- 支援閱後即焚

對於高度注重隱私的場景，Signal 是最佳選擇。

### 4.2 signal-cli 設定

```bash
# 安裝 signal-cli
# macOS
brew install signal-cli

# Linux
wget https://github.com/AsamK/signal-cli/releases/latest/download/signal-cli-linux.tar.gz
tar xf signal-cli-linux.tar.gz
sudo mv signal-cli /usr/local/bin/

# 註冊（需要一個可接收簡訊的電話號碼）
signal-cli -u +886912345678 register

# 驗證
signal-cli -u +886912345678 verify 123456

# 測試發送
signal-cli -u +886912345678 send -m "Hello from OpenClaw" +886923456789
```

### 4.3 OpenClaw Signal 配置

```json
{
  "channels": {
    "signal": {
      "enabled": true,
      "phoneNumber": "+886912345678",
      "signalCliPath": "/usr/local/bin/signal-cli",
      
      "allowFrom": {
        "users": [
          "signal:+886923456789"
        ]
      },
      
      "dmPolicy": {
        "enabled": true,
        "allowUnknownUsers": false,
        "disappearingMessages": {
          "enabled": true,
          "timer": 86400
        }
      },
      
      "privacy": {
        "noReadReceipts": true,
        "noTypingIndicator": true,
        "deleteAfterProcessing": true
      }
    }
  }
}
```

## 5. CLI 整合

### 5.1 終端機即時通訊

CLI 是 OpenClaw 最基本也最強大的通道 — 直接在終端機中與 AI 對話：

```bash
# 啟動 CLI 通道（互動模式）
openclaw chat

# 輸出範例：
# 🐾 EasonClaw v1.0.0
# Type your message, or /help for commands.
# 
# You: 今天天氣如何？
# EasonClaw: 讓我查一下...
# 
# 🌤️ 台北市目前天氣：
# 溫度：28°C
# 濕度：75%
# 狀況：多雲時晴
# 
# You: _
```

### 5.2 CLI 配置

```json
{
  "channels": {
    "cli": {
      "enabled": true,
      
      "interface": {
        "prompt": "You: ",
        "botPrefix": "EasonClaw: ",
        "colorEnabled": true,
        "theme": "dark",
        "showTimestamp": false,
        "showTokenCount": false
      },
      
      "features": {
        "multiline": true,
        "history": true,
        "historyFile": "~/.openclaw/cli-history",
        "autocomplete": true,
        "syntaxHighlight": true,
        "streaming": true
      },
      
      "commands": {
        "/help": "顯示使用說明",
        "/clear": "清除螢幕",
        "/reset": "重置對話記憶",
        "/export": "匯出對話紀錄",
        "/voice": "啟動語音模式",
        "/model": "切換 LLM 模型",
        "/debug": "切換除錯模式"
      }
    }
  }
}
```

### 5.3 進階 CLI 功能

**管線整合**：CLI 通道支援 Unix 管線，讓 OpenClaw 融入你的工作流：

```bash
# 將文件內容傳給 OpenClaw
cat error.log | openclaw ask "分析這個錯誤日誌"

# 將 OpenClaw 的回覆存檔
openclaw ask "幫我寫一個 Python 排序函數" > sort.py

# 搭配其他工具
git diff | openclaw ask "review 這個 PR"

# 批次處理
echo "翻譯成英文：你好世界" | openclaw --no-interactive
```

**非互動模式**：適合腳本和自動化：

```bash
# 單次問答
openclaw ask "2+2=?"

# 從文件讀取 prompt
openclaw ask --file prompt.txt

# JSON 輸出（方便程式處理）
openclaw ask --format json "列出 5 個 Python 框架"
```

## 6. Matrix 整合

### 6.1 Matrix 協議簡介

Matrix 是一個開放的、去中心化的通訊協議。任何人都可以架設自己的 Matrix 伺服器（Homeserver），實現完全自主控制的通訊。

### 6.2 OpenClaw Matrix 配置

```json
{
  "channels": {
    "matrix": {
      "enabled": true,
      "homeserverUrl": "https://matrix.example.com",
      "userId": "@easonclaw:example.com",
      "accessToken": "${MATRIX_ACCESS_TOKEN}",
      
      "allowFrom": {
        "users": ["matrix:@user:example.com"],
        "rooms": ["matrix:!roomid:example.com"]
      },
      
      "roomPolicy": {
        "autoJoin": false,
        "triggerMode": "mention",
        "encryption": true
      }
    }
  }
}
```

## 7. 自定義通道開發

### 7.1 通道 Adapter 介面

OpenClaw 的通道系統是可擴展的，你可以開發自定義通道 Adapter：

```typescript
interface ChannelAdapter {
  name: string;
  
  // 初始化
  init(config: ChannelConfig): Promise<void>;
  
  // 啟動（開始接收訊息）
  start(): Promise<void>;
  
  // 停止
  stop(): Promise<void>;
  
  // 發送訊息
  send(message: StandardMessage): Promise<void>;
  
  // 接收訊息的回調
  onMessage(callback: (message: StandardMessage) => void): void;
  
  // 健康檢查
  healthCheck(): Promise<boolean>;
}
```

### 7.2 開發範例

以下是一個簡單的自定義通道範例 — HTTP API 通道：

```typescript
// channels/http-api/adapter.ts
import { ChannelAdapter, StandardMessage, ChannelConfig } from '@openclaw/core';
import express from 'express';

export class HttpApiAdapter implements ChannelAdapter {
  name = 'http-api';
  private app: express.Application;
  private messageCallback: (msg: StandardMessage) => void;
  
  async init(config: ChannelConfig) {
    this.app = express();
    this.app.use(express.json());
    
    this.app.post('/api/message', (req, res) => {
      const msg: StandardMessage = {
        id: crypto.randomUUID(),
        channel: 'http-api',
        userId: req.body.userId,
        text: req.body.text,
        timestamp: Date.now()
      };
      
      this.messageCallback(msg);
      res.json({ status: 'received' });
    });
  }
  
  async start() {
    this.app.listen(3000);
  }
  
  async stop() { /* cleanup */ }
  
  async send(message: StandardMessage) {
    // 透過 WebSocket 或 SSE 推送回覆
  }
  
  onMessage(callback: (msg: StandardMessage) => void) {
    this.messageCallback = callback;
  }
  
  async healthCheck() { return true; }
}
```

### 7.3 發布自定義通道

自定義通道可以作為 ClawHub 擴展發布：

```yaml
# channel-http-api/CHANNEL.md
name: http-api
displayName: HTTP API Channel
version: 1.0.0
description: 透過 HTTP API 與 OpenClaw 互動
author: your-name
type: channel-adapter
```

## 8. 多通道同時運行

### 8.1 配置範例

OpenClaw 支援同時運行多個通道：

```json
{
  "channels": {
    "discord": { "enabled": true, "botToken": "${DISCORD_TOKEN}" },
    "telegram": { "enabled": true, "botToken": "${TELEGRAM_TOKEN}" },
    "cli": { "enabled": true },
    "whatsapp": { "enabled": false }
  }
}
```

### 8.2 跨通道注意事項

- **記憶隔離**：預設情況下，不同通道的對話記憶是隔離的
- **統一身份**：可以設定跨通道的使用者身份映射
- **通道偏好**：可以設定某些技能只在特定通道可用

```json
{
  "crossChannel": {
    "sharedMemory": false,
    "userMapping": {
      "eason": {
        "discord": "discord:123456",
        "telegram": "telegram:789012",
        "whatsapp": "whatsapp:+886912345678"
      }
    }
  }
}
```

## 9. 通道選擇指南

根據你的需求選擇最適合的通道：

| 需求 | 推薦通道 |
|------|----------|
| 團隊協作 | Slack |
| 社群互動 | Discord |
| 個人助手（亞洲） | WhatsApp / Telegram |
| 開發者工具 | CLI |
| 隱私優先 | Signal |
| Apple 生態圈 | iMessage |
| 自架部署 | Matrix |
| API 整合 | 自定義 HTTP 通道 |
| 多通道管理 | Discord + Telegram + CLI |

**EasonClaw 推薦的入門組合**：

1. **CLI**：開發和測試必備
2. **Discord**：主要互動通道，功能最完整
3. **Telegram**：輕量級備用通道

三個通道同時運行，覆蓋大部分使用場景。
