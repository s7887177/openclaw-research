# 測試框架與工具

## 本章摘要

本章介紹 OpenClaw 使用的測試框架、靜態分析工具、型別檢查系統與 Pre-commit Hooks。OpenClaw 作為一個大型 TypeScript 單體倉庫（Monorepo），採用 Vitest 作為測試框架，配合 TypeScript strict mode、oxlint、oxfmt 與 knip 構成完整的品質防線。

## 3. 測試共置模式（Co-location）

### 3.1 原則

OpenClaw 採用測試共置模式——`.test.ts` 檔案與其測試的源碼放在同一目錄：

```
src/process/
├── exec.ts
├── exec.no-output-timer.test.ts    # ← 測試與源碼同目錄
├── command-queue.ts
└── command-queue.test.ts           # ← 測試與源碼同目錄
```

### 3.2 各層級範例

**源碼層級（src/）：**
```
src/process/supervisor/supervisor.test.ts
src/process/supervisor/adapters/pty.test.ts
src/logger.test.ts
src/utils.test.ts
```

**Extension 層級（extensions/）：**
```
extensions/memory-wiki/src/lint.test.ts
extensions/memory-wiki/index.test.ts
extensions/memory-wiki/src/config-compat.test.ts
```

**Package 層級（packages/）：**
```
packages/plugin-package-contract/src/index.test.ts
packages/memory-host-sdk/src/host/*.test.ts
```

### 3.3 好處

- 容易找到對應的測試
- 修改源碼時自然會看到測試
- 支援 Vitest 的檔案監視模式
