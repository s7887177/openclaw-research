# Skill 開發與發佈

## 本章摘要

本章涵蓋 Skill 的開發流程、CLI 管理操作、測試方法與 ClawHub 發佈。從建立一個新 Skill 開始，到安裝驗證、最終發佈到社群市集的完整流程。

---

## 1. 建立新 Skill

### 1.1 最小可行 Skill

建立一個新 Skill 只需要兩步：

```bash
# 1. 建立目錄
mkdir -p ~/.openclaw/workspace/skills/my-skill

# 2. 建立 SKILL.md
cat > ~/.openclaw/workspace/skills/my-skill/SKILL.md << 'EOF'
---
name: my-skill
description: "My custom skill that does amazing things"
---

# My Skill

## 使用方式

當使用者需要做某件事時，使用以下步驟：

1. 第一步...
2. 第二步...
3. 第三步...

## 注意事項

- 重要的限制或規則
EOF
```

### 1.2 帶需求的 Skill

如果 Skill 需要外部工具：

```markdown
---
name: my-tool-skill
description: "Uses external tool X for processing"
metadata:
  {
    "openclaw": {
      "emoji": "🔧",
      "requires": {
        "bins": ["mytool"]
      },
      "install": [
        {
          "id": "brew-mytool",
          "kind": "brew",
          "formula": "mytool",
          "bins": ["mytool"],
          "label": "Install mytool via Homebrew"
        },
        {
          "id": "npm-mytool",
          "kind": "node",
          "package": "mytool",
          "bins": ["mytool"],
          "label": "Install mytool via npm"
        }
      ]
    }
  }
---

# My Tool Skill

使用 `mytool` 命令來處理...
```

### 1.3 帶腳本的 Skill

Skill 可以包含腳本，AI 會根據指引呼叫這些腳本：

```
my-skill/
├── SKILL.md
└── scripts/
    └── process.py
```

在 SKILL.md 中引用腳本：

```markdown
## 使用腳本

執行處理：
\```bash
python {baseDir}/scripts/process.py --input data.csv
\```

`{baseDir}` 會被自動替換為 Skill 的目錄路徑。
```

## 5. ClawHub 發佈

### 5.1 ClawHub 概述

ClawHub 是 OpenClaw 的 Skill 市集，讓開發者可以分享自訂 Skill。

### 5.2 ClawHub CLI

ClawHub Skill 提供了市集操作的 CLI 指引：

```bash
# 登入
clawhub login
clawhub whoami

# 瀏覽與安裝
clawhub list
clawhub install my-skill [--version <version>]
clawhub update [my-skill] [--all] [--force]

# 發佈
clawhub publish ./my-skill \
  --slug my-skill \
  --name "My Skill" \
  --version 1.2.0
```
> 引用：`source-repo/skills/clawhub/SKILL.md:9-25`

## 6. 測試 Skill

### 6.1 手動測試

最直接的測試方式是將 Skill 放到工作區中，然後與 AI 互動：

```bash
# 1. 確保 Skill 在工作區中
ls ~/.openclaw/workspace/skills/my-skill/SKILL.md

# 2. 檢查 Skill 是否被載入
openclaw skills list

# 3. 檢查依賴是否就緒
openclaw skills check

# 4. 與 AI 互動測試
openclaw agent --message "使用 my-skill 做某件事"
```

### 6.2 資格檢查

確認 Skill 的 `requires` 條件是否滿足：

```bash
openclaw skills check --json | jq '.[] | select(.name == "my-skill")'
```

### 6.3 監視模式

開發時可以啟用 Skill 監視，讓變更自動重載：

```json
{
  "skills": {
    "load": {
      "watch": true,
      "watchDebounceMs": 500
    }
  }
}
```

## 8. 開發最佳實踐

### 8.1 命名慣例

- 使用 kebab-case：`my-awesome-skill`
- 名稱應描述功能：`github-pr-review` 而非 `pr-tool`
- 避免與內建 Skill 衝突

### 8.2 SKILL.md 撰寫原則

1. **開頭說明用途**：第一段說清楚 Skill 做什麼
2. **提供具體命令**：包含實際可執行的命令範例
3. **說明限制**：明確列出不能做的事
4. **使用 `{baseDir}`**：引用 Skill 目錄中的資源
5. **保持精簡**：受 `maxSkillFileBytes` 限制

### 8.3 安全注意事項

- Skill 中的腳本會被 AI 執行——確保腳本是安全的
- 避免在 Skill 中硬編碼敏感資訊
- 使用 `apiKey` 配置欄位處理金鑰：

```json
{
  "skills": {
    "entries": {
      "my-skill": {
        "apiKey": { "env": "MY_SKILL_API_KEY" }
      }
    }
  }
}
```
> 引用：`source-repo/src/config/types.skills.ts:3-8`
