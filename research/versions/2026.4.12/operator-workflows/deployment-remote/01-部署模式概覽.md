# 部署模式概覽

## 本章摘要

OpenClaw 提供多種部署方式以適應不同場景：從最簡單的本地 CLI 執行，到 Daemon 背景服務，再到 Docker/Podman 容器化部署與雲端平台。本章詳細說明每種模式的架構、配置方法與適用場景。

---

## 1. 部署模式總覽

| 模式 | 適用場景 | 持久性 | 配置複雜度 |
|------|---------|-------|-----------|
| 本地 CLI | 快速測試、開發 | 手動啟動 | 最低 |
| Daemon 模式 | 個人日常使用 | 系統服務自動啟動 | 低 |
| Docker 模式 | 伺服器部署、隔離 | 容器重啟策略 | 中 |
| Podman 模式 | 無 root 容器部署 | Quadlet 服務 | 中 |
| 雲端部署 | 永遠在線、多裝置 | 平台管理 | 高 |

---

## 2. 本地 CLI 模式

最基本的使用方式，直接在終端機啟動 Gateway：

```bash
openclaw gateway --port 18789 --verbose
```
> 引用：`source-repo/README.md:121`

此模式下 Gateway 綁定到 loopback（`127.0.0.1`），僅本機可存取。關閉終端機後 Gateway 停止。

### 2.1 Gateway 預設埠號

Gateway 有一組預設的埠號配置：

| 埠號 | 常數 | 用途 |
|------|------|------|
| 18789 | `DEFAULT_GATEWAY_PORT` | Gateway WebSocket/HTTP |
| 18790 | `DEFAULT_BRIDGE_PORT` | Bridge 伺服器 |
| 18791 | `DEFAULT_BROWSER_CONTROL_PORT` | 瀏覽器控制 |
| 18793 | `DEFAULT_CANVAS_HOST_PORT` | Canvas 主機 |
| 18800-18899 | `DEFAULT_BROWSER_CDP_*` | 瀏覽器 CDP 範圍 |

> 引用：`source-repo/src/config/port-defaults.ts:15-19`

衍生埠號由 Gateway 主埠計算而來：

```typescript
deriveDefaultBridgePort(gatewayPort)         // gatewayPort + 1
deriveDefaultBrowserControlPort(gatewayPort)  // gatewayPort + 2
deriveDefaultCanvasHostPort(gatewayPort)      // gatewayPort + 4
```
> 引用：`source-repo/src/config/port-defaults.ts:23-44`

### 2.2 Bind 模式

Gateway 支援五種綁定模式（Bind Mode），控制監聽的網路介面：

```typescript
VALID_BIND_MODES = new Set<string>(["loopback", "lan", "auto", "custom", "tailnet"])
```
> 引用：`source-repo/src/cli/gateway-cli/run.ts:317`

| 模式 | 綁定位址 | 說明 |
|------|---------|------|
| `loopback` | `127.0.0.1` | 僅本機（安全預設值）|
| `lan` | `0.0.0.0` | 所有介面（區域網路可存取）|
| `auto` | 自動 | 容器環境中自動選擇 |
| `custom` | 自訂 | 使用 `cfg.gateway.customBindHost` |
| `tailnet` | Tailscale 網路 | 綁定到 Tailscale 介面 |

> 引用：`source-repo/src/cli/gateway-cli/run.ts:359-364`

CLI 選項：

```bash
openclaw gateway --bind lan       # 監聽所有介面
openclaw gateway --bind loopback  # 僅本機
openclaw gateway --bind custom    # 使用配置中的自訂位址
```
> 引用：`source-repo/src/cli/gateway-cli/run.ts:602-604`

## 3. Daemon 模式

Daemon 模式讓 Gateway 作為系統服務在背景持續運行，是推薦的個人使用方式。

### 3.1 安裝 Daemon

```bash
openclaw onboard --install-daemon
# 或單獨安裝
openclaw daemon install
```

### 3.2 Daemon CLI 操作

```bash
openclaw daemon start      # 啟動
openclaw daemon stop       # 停止
openclaw daemon status     # 查看狀態
openclaw daemon install    # 安裝為系統服務
openclaw daemon uninstall  # 移除系統服務
```
> 引用：`source-repo/src/cli/daemon-cli.ts:4-9`

文件連結：`docs.openclaw.ai/cli/gateway`

> 引用：`source-repo/src/cli/daemon-cli/register.ts:6-14`

### 3.3 平台實作

| 平台 | 服務機制 | 啟動生命週期 |
|------|---------|------------|
| macOS | `launchd` (LaunchAgent) | 使用者登入時自動啟動 |
| Linux | `systemd` (user service) | 需要 linger 支援 |
| Windows | `schtasks` (排程工作) | 使用者登入時觸發 |

> 引用：`source-repo/src/daemon/launchd.ts`, `source-repo/src/daemon/systemd.ts`, `source-repo/src/daemon/schtasks.ts`

### 3.4 Daemon 啟動命令

Daemon 啟動的 Gateway 使用 `lifecycle.ts` 中的函式管理生命週期：

```typescript
async function runDaemonStart(opts: DaemonLifecycleOptions)  // Line 145
async function runDaemonStop(opts: DaemonLifecycleOptions)   // Line 158
```
> 引用：`source-repo/src/cli/daemon-cli/lifecycle.ts:145, 158`

## 4. Docker 模式

### 4.1 Docker Compose 配置

OpenClaw 提供完整的 `docker-compose.yml`，定義了兩個服務：

#### Gateway 服務

```yaml
services:
  openclaw-gateway:
    image: ${OPENCLAW_IMAGE:-openclaw:local}
    environment:
      HOME: /home/node
      TERM: xterm-256color
      OPENCLAW_GATEWAY_TOKEN: ${OPENCLAW_GATEWAY_TOKEN:-}
      TZ: ${OPENCLAW_TZ:-UTC}
    volumes:
      - ${OPENCLAW_CONFIG_DIR}:/home/node/.openclaw
      - ${OPENCLAW_WORKSPACE_DIR}:/home/node/.openclaw/workspace
    ports:
      - "${OPENCLAW_GATEWAY_PORT:-18789}:18789"
      - "${OPENCLAW_BRIDGE_PORT:-18790}:18790"
    init: true
    restart: unless-stopped
    command: ["node", "dist/index.js", "gateway", "--bind", "${OPENCLAW_GATEWAY_BIND:-lan}", "--port", "18789"]
```
> 引用：`source-repo/docker-compose.yml:1-38`

#### CLI 服務

```yaml
  openclaw-cli:
    image: ${OPENCLAW_IMAGE:-openclaw:local}
    network_mode: "service:openclaw-gateway"
    cap_drop:
      - NET_RAW
      - NET_ADMIN
    security_opt:
      - no-new-privileges:true
    stdin_open: true
    tty: true
    init: true
    entrypoint: ["node", "dist/index.js"]
    depends_on:
      - openclaw-gateway
```
> 引用：`source-repo/docker-compose.yml:52-79`

注意 CLI 服務的安全強化：
- 丟棄 `NET_RAW` 和 `NET_ADMIN` 能力
- 設定 `no-new-privileges:true`
- 使用 Gateway 的網路命名空間

#### 健康檢查

Docker Compose 中的 Gateway 健康檢查：

```yaml
healthcheck:
  test: ["CMD", "node", "-e", "fetch('http://127.0.0.1:18789/healthz').then((r)=>process.exit(r.ok?0:1)).catch(()=>process.exit(1))"]
  interval: 30s
  timeout: 5s
  retries: 5
  start_period: 20s
```
> 引用：`source-repo/docker-compose.yml:39-50`

### 4.2 Dockerfile 架構

Dockerfile 使用多階段建構（Multi-stage Build），產出最小的執行時映像：

```
Stage 1: ext-deps     → 提取 Extension 的 package.json
Stage 2: build        → 完整建構（安裝依賴、建構、打包）
Stage 3: runtime-assets → 刪除開發依賴
Stage 4: runtime      → 最終執行時映像
```
> 引用：`source-repo/Dockerfile:1-277`

關鍵特性：

**兩種映像變體：**
```dockerfile
# 預設（bookworm）：docker build .
# 精簡（bookworm-slim）：docker build --build-arg OPENCLAW_VARIANT=slim .
```
> 引用：`source-repo/Dockerfile:13-14`

**可選建構參數：**

| 參數 | 用途 | 說明 |
|------|------|------|
| `OPENCLAW_EXTENSIONS` | 啟用的 Extension | 空格分隔，如 `"diagnostics-otel matrix"` |
| `OPENCLAW_VARIANT` | 映像變體 | `default` 或 `slim` |
| `OPENCLAW_INSTALL_BROWSER` | 安裝 Chromium | 加 ~300MB，免去每次啟動的安裝 |
| `OPENCLAW_INSTALL_DOCKER_CLI` | 安裝 Docker CLI | Sandbox 功能所需 |
| `OPENCLAW_DOCKER_APT_PACKAGES` | 額外 APT 套件 | 如 `"python3 wget"` |

> 引用：`source-repo/Dockerfile:4-5, 15-16, 194-195, 204-207, 220-221`

**安全強化：**
```dockerfile
# 以非 root 使用者執行
USER node
```
> 引用：`source-repo/Dockerfile:257-259`

**預設啟動命令：**
```dockerfile
CMD ["node", "openclaw.mjs", "gateway", "--allow-unconfigured"]
```
> 引用：`source-repo/Dockerfile:276`

**Dockerfile 內建健康檢查：**
```dockerfile
HEALTHCHECK --interval=3m --timeout=10s --start-period=15s --retries=3 \
  CMD node -e "fetch('http://127.0.0.1:18789/healthz').then((r)=>process.exit(r.ok?0:1)).catch(()=>process.exit(1))"
```
> 引用：`source-repo/Dockerfile:274-275`

### 4.3 Docker 建構與啟動

```bash
# 基本建構
docker build -t openclaw:local .

# 帶 Extension 的建構
docker build --build-arg OPENCLAW_EXTENSIONS="diagnostics-otel matrix" -t openclaw:local .

# 啟動（使用 docker-compose）
OPENCLAW_CONFIG_DIR=~/.openclaw \
OPENCLAW_WORKSPACE_DIR=~/.openclaw/workspace \
OPENCLAW_GATEWAY_TOKEN=$(openssl rand -hex 32) \
docker compose up -d
```

### 4.4 Sandbox 支援

Docker 中的 Sandbox（沙箱隔離）需要額外設定：

```yaml
## 取消註解以啟用 Sandbox 隔離
# - /var/run/docker.sock:/var/run/docker.sock
```
> 引用：`source-repo/docker-compose.yml:16-22`

需要：
1. 建構時加入 `--build-arg OPENCLAW_INSTALL_DOCKER_CLI=1`
2. 或使用 `scripts/docker/setup.sh` 配合 `OPENCLAW_SANDBOX=1`
3. 設定 `DOCKER_GID` 為主機的 docker 群組 GID

### 5.1 配置範本

```bash

OPENCLAW_GATEWAY_BIND=lan
```

## 6. 雲端部署

### 6.1 Fly.io 部署

OpenClaw 提供 `fly.toml` 配置：

```toml
app = "openclaw"
primary_region = "iad"

[build]
  dockerfile = "Dockerfile"

[env]
  NODE_ENV = "production"
  OPENCLAW_PREFER_PNPM = "1"
  OPENCLAW_STATE_DIR = "/data"
  NODE_OPTIONS = "--max-old-space-size=1536"

[processes]
  app = "node dist/index.js gateway --allow-unconfigured --port 3000 --bind lan"
```
> 引用：`source-repo/fly.toml:4-5, 8, 10-15, 18`

```toml
[[services]]
  internal_port = 3000
  protocol = "tcp"
  [services.http_checks]
    force_https = true
  auto_stop_machines = false

[[vm]]
  size = "shared-cpu-2x"
  memory = "2048mb"

[[mounts]]
  source = "openclaw_data"
  destination = "/data"
```
> 引用：`source-repo/fly.toml:21-35`

### 6.2 Render 部署

OpenClaw 提供 `render.yaml` 配置：

```yaml
services:
  - type: web
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: OPENCLAW_GATEWAY_PORT
        value: "8080"
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true
    disk:
      name: openclaw-data
      sizeGB: 1
      mountPath: /data
```
> 引用：`source-repo/render.yaml:1-19`

### 6.3 各環境 Gateway 啟動命令比較

| 環境 | 命令 | Bind | 埠號 | 認證 |
|------|------|------|------|------|
| Dockerfile 預設 | `gateway --allow-unconfigured` | loopback | 18789 | 無 |
| Docker Compose | `gateway --bind lan --port 18789` | lan | 18789 | Token/環境變數 |
| Fly.io | `gateway --allow-unconfigured --port 3000 --bind lan` | lan | 3000 | 無 |
| Render | Docker 預設 + 環境變數 | lan | 8080 | Token（自動生成）|
| 本地 CLI | `openclaw gateway run` | loopback | 18789 | 配置驅動 |
