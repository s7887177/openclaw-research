# 遠端存取與安全

## 本章摘要

本章說明如何從遠端存取 OpenClaw Gateway，包括 Tailscale tunnel、SSH tunnel 與直接暴露埠。同時深入介紹 Gateway 的認證機制（Token/Password/Device Pairing）與裝置管理。

---

## 1. Gateway 認證模式

Gateway 支援多種認證模式，定義在 `auth.ts` 中：

```typescript
type ResolvedGatewayAuthMode = "none" | "token" | "password" | "trusted-proxy";
```
> 引用：`source-repo/src/gateway/auth.ts:29`

### 1.1 認證方式總覽

| 模式 | 說明 | 適用場景 |
|------|------|---------|
| `none` | 無認證 | 本地開發、loopback 綁定 |
| `token` | 共享 Token | Docker/雲端部署 |
| `password` | 密碼認證 | 個人遠端存取 |
| `trusted-proxy` | 信任的反向代理 | 在負載均衡器後方 |

### 1.2 認證方法細分

Gateway 在驗證請求時，會辨識更細緻的認證方法：

```typescript
type GatewayAuthResult = {
  method: "none" | "token" | "password" | "tailscale" | "device-token" | "bootstrap-token" | "trusted-proxy";
  // ...
}
```
> 引用：`source-repo/src/gateway/auth.ts:51-67`

這些方法包括：
- **token**：直接的共享 Token
- **password**：密碼驗證
- **tailscale**：來自 Tailscale 網路的請求
- **device-token**：已配對裝置的 Token
- **bootstrap-token**：首次配對用的臨時 Token
- **trusted-proxy**：透過信任代理轉發的請求

### 1.3 認證模式衝突偵測

系統會偵測同時設定 Token 和 Password 的衝突情況：

```typescript
// auth-mode-policy.ts
// hasAmbiguousGatewayAuthModeConfig() - 偵測模糊的認證配置
// assertExplicitGatewayAuthModeWhenBothConfigured() - 驗證
```
> 引用：`source-repo/src/gateway/auth-mode-policy.ts:4-19`

### 1.4 Gateway Token 設定

透過 CLI 或環境變數設定 Token：

```bash
# CLI 參數
openclaw gateway --token <your-token>

# 環境變數
export OPENCLAW_GATEWAY_TOKEN=your-token-here
```
> 引用：`source-repo/src/cli/gateway-cli/run.ts:606-608`, `source-repo/docker-compose.yml:7`

## 2. Tailscale 整合

### 2.1 Tailscale 模式

OpenClaw 原生整合 Tailscale，提供安全的遠端存取：

```typescript
async function startGatewayTailscaleExposure(params: {
  tailscaleMode: "off" | "serve" | "funnel";
  resetOnExit?: boolean;
  port: number;
  controlUiBasePath?: string;
  logTailscale: { info, warn };
}): Promise<(() => Promise<void>) | null>
```
> 引用：`source-repo/src/gateway/server-tailscale.ts:1-57`

三種 Tailscale 模式：

| 模式 | 命令 | 說明 |
|------|------|------|
| `off` | 不使用 Tailscale | 預設，返回 `null` |
| `serve` | `enableTailscaleServe(port)` | 在 Tailscale 網路內提供服務 |
| `funnel` | `enableTailscaleFunnel(port)` | 透過網際網路公開服務 |

> 引用：`source-repo/src/gateway/server-tailscale.ts:22-26`

## 3. SSH Tunnel

對於不使用 Tailscale 的場景，可以使用 SSH tunnel：

```bash
# 從遠端機器建立 SSH tunnel
ssh -L 18789:localhost:18789 user@your-server

# 然後在本地存取
# http://localhost:18789
```

這個方法利用 Gateway 預設的 loopback 綁定，不需要額外的認證配置——因為 SSH 本身已提供安全通道。

## 7. 多裝置共用 Gateway

### 7.1 架構模式

多裝置共用同一個 Gateway 的推薦架構：

```
手機 (iOS/Android) ──┐
                     ├── Tailscale / 區域網路 ──→ Gateway (一台主機)
桌面 (macOS/Linux) ──┤                           ├── LLM Provider (OpenAI/Anthropic)
                     │                           ├── Channel (Discord/Telegram/...)
平板 ────────────────┘                           └── Memory (本地記憶)
```

### 7.2 安全建議

| 配置 | 僅本機 | 區域網路 | 網際網路 |
|------|-------|---------|---------|
| Bind 模式 | `loopback` | `lan` | `lan` + 反向代理 |
| 認證 | `none` | `token` 或 `password` | `token` + TLS |
| Tailscale | 不需要 | 可選 | 建議 `funnel` |
| 額外安全 | — | — | 強烈建議 TLS |

### 7.3 連線安全環境變數

```bash
# 允許不安全的私有 WebSocket 連線（僅供開發）
OPENCLAW_ALLOW_INSECURE_PRIVATE_WS=1
```
> 引用：`source-repo/docker-compose.yml:8, 64`

⚠️ **警告**：此選項僅供開發使用，生產環境請勿啟用。
