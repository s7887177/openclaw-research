# 系統需求與安裝

## 本章摘要

本章從零開始介紹 OpenClaw 的系統需求、安裝方式、配置檔結構與位置。OpenClaw 是一個以 Node.js 為執行環境的全域 CLI 工具，透過 npm 或 pnpm 安裝後，以 `openclaw` 命令操作。所有使用者配置儲存於 `~/.openclaw/` 目錄下。

---

## 1. 系統需求

### 1.1 Node.js 版本

OpenClaw 對 Node.js 版本有明確要求。在 `package.json` 的 `engines` 欄位中定義：

```json
"engines": {
  "node": ">=22.14.0"
}
```
> 引用：`source-repo/package.json:1472-1474`

而在 `README.md` 中，官方推薦更高的版本：

> Runtime: **Node 24 (recommended) or Node 22.16+**.

> 引用：`source-repo/README.md:101`

也就是說：

| 層級 | Node 版本 | 說明 |
|------|-----------|------|
| 最低要求 | ≥ 22.14.0 | `engines` 欄位定義，npm install 時檢查 |
| 建議版本 | ≥ 22.16 | 文件推薦，確保所有功能穩定 |
| 推薦版本 | Node 24 | 最佳效能與相容性 |

為什麼需要 Node 22+？因為 OpenClaw 大量使用了 ES2023 特性（如 `Array.prototype.findLast`、`import.meta.url`）以及 Node.js 原生的 `node:test`、`node:fs/promises` 等模組。Dockerfile 中更是直接使用 Node 24 映像檔：

```dockerfile
ARG OPENCLAW_NODE_BOOKWORM_IMAGE="node:24-bookworm@sha256:..."
```
> 引用：`source-repo/Dockerfile:19`

### 1.2 套件管理器

OpenClaw 專案本身使用 pnpm 作為套件管理器：

```json
"packageManager": "pnpm@10.32.1"
```
> 引用：`source-repo/package.json:1475`

但對終端使用者而言，安裝時支援三種套件管理器：

- **npm**：`npm install -g openclaw@latest`
- **pnpm**：`pnpm add -g openclaw@latest`
- **bun**：也受支援

> 引用：`source-repo/README.md:103-105`

### 1.3 支援的作業系統

OpenClaw 支援 macOS、Linux 與 Windows（透過 WSL2）：

> OpenClaw Onboard guides you step by step ... works on **macOS, Linux, and Windows (via WSL2; strongly recommended)**.

> 引用：`source-repo/README.md:29`

不同作業系統的 Daemon 實作方式不同：

| 作業系統 | Daemon 機制 | 實作檔案 |
|----------|------------|---------|
| macOS | launchd | `source-repo/src/daemon/launchd.ts` |
| Linux | systemd (user service) | `source-repo/src/daemon/systemd.ts` |
| Windows | schtasks | `source-repo/src/daemon/schtasks.ts` |

### 1.4 Nix 模式

OpenClaw 支援 Nix 套件管理器的特殊模式。透過環境變數 `OPENCLAW_NIX_MODE=1` 啟用：

```typescript
export function resolveIsNixMode(env: NodeJS.ProcessEnv = process.env): boolean {
  return env.OPENCLAW_NIX_MODE === "1";
}
```
> 引用：`source-repo/src/config/paths.ts:14-16`

在 Nix 模式下：
- 不會嘗試自動安裝流程（No auto-install flows）
- 缺少的依賴會產生 Nix 專用的錯誤訊息
- 配置由 Nix 外部管理（唯讀）

> 引用：`source-repo/src/config/paths.ts:7-13`

Nix 安裝使用獨立的外部倉庫：[https://github.com/openclaw/nix-openclaw](https://github.com/openclaw/nix-openclaw)

> 引用：`source-repo/README.md:26`

## 2. 安裝方式

### 2.1 全域安裝

OpenClaw 是一個 npm 套件，安裝後會在系統中註冊 `openclaw` 命令：

```json
{
  "name": "openclaw",
  "version": "2026.4.12",
  "bin": {
    "openclaw": "openclaw.mjs"
  }
}
```
> 引用：`source-repo/package.json:2, 3, 16-18`

標準安裝命令：

```bash
npm install -g openclaw@latest
# 或
pnpm add -g openclaw@latest
```

安裝完成後，`openclaw` 命令指向 `openclaw.mjs` 這個入口檔案。

### 2.2 安裝後的首次啟動

安裝後推薦的第一個命令是：

```bash
openclaw onboard --install-daemon
```
> 引用：`source-repo/README.md:107`

這個命令會：
1. 啟動互動式設定精靈（Setup Wizard）
2. 引導選擇 LLM Provider 並設定認證
3. 設定 Channel（通訊頻道）
4. 安裝 Gateway Daemon（讓 Gateway 在背景持續運行）

> OpenClaw Onboard installs the Gateway daemon (launchd/systemd user service) so it stays running.

> 引用：`source-repo/README.md:110`

### 2.3 安裝包含的檔案

npm 套件中包含以下檔案（由 `files` 欄位定義）：

```json
"files": [
  "CHANGELOG.md",
  "LICENSE",
  "openclaw.mjs",
  "README.md",
  "assets/",
  "dist/",
  ...
]
```
> 引用：`source-repo/package.json:23-30`

核心結構：
- `openclaw.mjs`：CLI 入口點
- `dist/`：編譯後的 JavaScript（排除 `.map` 檔案）
- `assets/`：靜態資源
- `skills/`：內建 Skill 定義
- `docs/`：內嵌文件
- `extensions/`：內建 Plugin/Extension

## 3. 配置檔位置與格式

### 3.1 狀態目錄

OpenClaw 的所有資料儲存在使用者家目錄下的 `.openclaw/` 資料夾：

```typescript
const NEW_STATE_DIRNAME = ".openclaw";
const CONFIG_FILENAME = "openclaw.json";
```
> 引用：`source-repo/src/config/paths.ts:22-23`

完整路徑為 `~/.openclaw/`。這個目錄包含：

| 路徑 | 用途 |
|------|------|
| `~/.openclaw/openclaw.json` | 主配置檔 |
| `~/.openclaw/agents/` | Agent 會話資料、Session 記錄 |
| `~/.openclaw/workspace/` | 工作區（技能、記憶等） |
| `~/.openclaw/plugins/` | 已安裝的 Plugin |

### 3.2 Legacy 目錄支援

OpenClaw 經歷過品牌重塑（rebrand），因此仍支援舊版目錄名稱：

```typescript
const LEGACY_STATE_DIRNAMES = [".clawdbot"] as const;
const LEGACY_CONFIG_FILENAMES = ["clawdbot.json"] as const;
```
> 引用：`source-repo/src/config/paths.ts:21, 24`

系統會自動偵測並遷移舊版配置。

### 3.3 配置檔格式

主配置檔為 `openclaw.json`（JSON 格式），包含以下主要區塊：

- **providers**：LLM Provider 認證設定
- **channels**：通訊頻道設定（Discord、Telegram、WhatsApp 等）
- **gateway**：Gateway 伺服器設定
- **agents**：Agent 行為設定
- **plugins**：Plugin 啟用/停用/配置
- **skills**：Skill 配置
- **hooks**：Hook 設定
- **cron**：排程任務設定
- **memory**：記憶系統設定
- **talk**：語音對話設定
- **mcp**：MCP（Model Context Protocol）設定

### 3.4 環境變數覆蓋

配置值可以透過環境變數覆蓋。OpenClaw 支援環境變數替換（Environment Variable Substitution）：

```typescript
// src/config/env-substitution.ts
```
> 引用：`source-repo/src/config/env-substitution.ts`

以及 `.env` 檔案的載入：

```typescript
// src/cli/dotenv.ts
```
> 引用：`source-repo/src/cli/dotenv.ts`

### 3.5 配置檔 CLI 管理

透過 `openclaw config` 命令管理配置：

```
openclaw config set <path> <value>    # 設定值
openclaw config get <path>            # 讀取值
```

相關實作在：
- `source-repo/src/cli/config-cli.ts` — 配置 CLI 命令
- `source-repo/src/config/config-paths.ts` — 路徑解析（`parseConfigPath()`, `setConfigValueAtPath()`, `getConfigValueAtPath()`）

> 引用：`source-repo/src/cli/config-cli.ts`, `source-repo/src/config/config-paths.ts:1-83`

## 4. 安裝驗證

安裝完成後，可以透過以下方式驗證：

### 4.1 版本確認

```bash
openclaw --version
# 輸出: 2026.4.12
```

版本號定義在 `package.json` 中：

```json
"version": "2026.4.12"
```
> 引用：`source-repo/package.json:3`

### 4.2 Doctor 快速檢查

```bash
openclaw doctor
```

這會執行完整的健康檢查（詳見下一章）。

### 4.3 Gateway 測試啟動

```bash
openclaw gateway --port 18789 --verbose
```
> 引用：`source-repo/README.md:121`

Gateway 預設監聽 18789 埠：

```typescript
export const DEFAULT_GATEWAY_PORT = 18789;
```
> 引用：`source-repo/src/config/paths.ts:214`（此為 explore agent 回報的行號，路徑可能在 `port-defaults.ts` 中）

## 5. 快速上手流程

完整的從零到一流程：

```bash
# 1. 安裝 Node.js 24（推薦）
# 使用 nvm, fnm, 或系統套件管理器

# 2. 安裝 OpenClaw
npm install -g openclaw@latest

# 3. 首次設定（互動式精靈 + 安裝 Daemon）
openclaw onboard --install-daemon

# 4. 驗證
openclaw doctor

# 5. 啟動 Gateway（如果 Daemon 未安裝或想手動啟動）
openclaw gateway --port 18789 --verbose

# 6. 發送測試訊息
openclaw agent --message "Hello from OpenClaw"
```

> 引用：`source-repo/README.md:99-128`
