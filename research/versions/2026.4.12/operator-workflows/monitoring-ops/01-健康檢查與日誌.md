# 健康檢查與日誌

## 本章摘要

本章深入介紹 OpenClaw Gateway 的健康檢查端點、日誌系統架構、WebSocket 通訊日誌、Channel 健康監控器與 Gateway 維護計時器。這些組件共同構成了 OpenClaw 的運維可觀測性基礎。

---

### 1.3 兩種探測的區別

| 探測 | 端點 | 含義 | 失敗影響 |
|------|------|------|---------|
| Liveness（存活）| `/health`, `/healthz` | Gateway 程序正在運行 | 容器被重啟 |
| Readiness（就緒）| `/ready`, `/readyz` | Gateway 已準備好處理請求 | 暫時停止接收流量 |

### 1.4 Docker 健康檢查

Dockerfile 中的健康檢查使用 `fetch` API 呼叫 `/healthz`：

```dockerfile
HEALTHCHECK --interval=3m --timeout=10s --start-period=15s --retries=3 \
  CMD node -e "fetch('http://127.0.0.1:18789/healthz').then((r)=>process.exit(r.ok?0:1)).catch(()=>process.exit(1))"
```
> 引用：`source-repo/Dockerfile:274-275`

Docker Compose 中使用更積極的檢查間隔：

```yaml
healthcheck:
  test: ["CMD", "node", "-e", "fetch('http://127.0.0.1:18789/healthz')..."]
  interval: 30s
  timeout: 5s
  retries: 5
  start_period: 20s
```
> 引用：`source-repo/docker-compose.yml:39-50`

健康評估的原因（Reason）：

| 原因 | 說明 |
|------|------|
| `healthy` | 正常 |
| `unmanaged` | 不受管理 |

| `stuck` | 卡住 |
| `startup-connect-grace` | 啟動連線寬限期中 |
| `disconnected` | 斷線 |
| `stale-socket` | 過時的 Socket |
