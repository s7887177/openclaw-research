# Provider 實作與參考

## 本章摘要

本章深入介紹 Provider 的實作細節，包括 HTTP 處理工具、串流（Streaming）實作、參考 Provider 的架構分析，以及外部 Plugin 的驗證契約。從 OpenAI、Anthropic、Ollama 三個官方 Provider 中提取可複用的實作模式。

## 3. Provider 註冊 API

### 3.1 registerProvider() 完整參數

Provider 透過 `api.registerProvider()` 註冊，參數結構非常豐富：

```typescript
api.registerProvider({
  // 基本資訊
  id: string,              // Provider ID
  label: string,           // 顯示名稱
  docsPath?: string,       // 文件路徑
  aliases?: string[],      // 使用者面向的別名
  hookAliases?: string[],  // 內部 Hook 別名
  envVars?: string[],      // 認證環境變數

  // 認證
  auth: ProviderAuthMethod[],

  // 模型發現
  catalog?: ProviderPluginCatalog,
  discovery?: ProviderPluginDiscovery,

  // 模型處理 Hook
  resolveDynamicModel?: hook,      // 動態模型解析
  prepareDynamicModel?: hook,      // 非同步模型準備
  normalizeResolvedModel?: hook,   // 傳輸正規化
  normalizeConfig?: hook,          // 配置正規化
  normalizeTransport?: hook,       // 傳輸家族正規化
  normalizeModelId?: hook,         // 模型 ID 清理

  // 串流 Hook
  wrapStreamFn?: hook,             // 串流包裝

  // 重播 Hook
  buildReplayPolicy?: hook,        // 重播政策
  sanitizeReplayHistory?: hook,    // 對話記錄清理

  // 模型屬性 Hook
  isModernModelRef?: hook,         // 現代模型判定
  resolveDefaultThinkingLevel?: hook, // 預設思考等級

  // 認證 Hook
  resolveSyntheticAuth?: hook,     // 合成認證
  resolveUsageAuth?: hook,         // 使用量 API 認證

  // 使用量 Hook
  fetchUsageSnapshot?: hook,       // 取得使用量快照

  // 快取 Hook
  isCacheTtlEligible?: hook,      // 快取資格判定

  // 診斷 Hook
  buildAuthDoctorHint?: hook,     // Doctor 診斷提示
});
```
> 引用：`source-repo/extensions/anthropic/register.runtime.ts:380-496`

## 6. Provider 開發步驟

### 6.1 建立新 Provider 的步驟

```bash
# 1. 建立 Extension 目錄
mkdir -p extensions/my-provider/src

# 2. 建立 openclaw.plugin.json
# 3. 建立 index.ts (主入口)
# 4. 實作 register() 回呼
# 5. 實作認證方法
# 6. 實作串流函式
# 7. 建立 package.json (含 openclaw 相容性欄位)
```

### 6.2 最小可行 Provider

一個最小的 Provider 需要：

1. **`openclaw.plugin.json`**：宣告 Provider ID、模型前綴、認證環境變數
2. **`index.ts`**：匯出 `definePluginEntry()` 或 `defineSingleProviderPluginEntry()`
3. **註冊函式**：呼叫 `api.registerProvider()` 並提供 `auth` 和基本的 hook

```typescript
// Plugin SDK 提供的 helper：
import { definePluginEntry, defineSingleProviderPluginEntry } from "openclaw/plugin-sdk/provider-entry";
```
> 引用：`source-repo/src/plugin-sdk/provider-entry.ts:1-168`

### 6.3 測試 Provider

```bash
# 1. 將 Extension 放到 extensions/ 目錄
# 2. 在 openclaw.json 中啟用
# 3. 啟動 Gateway
openclaw gateway --verbose

# 4. 測試模型回應
openclaw agent --model my-model --message "Hello"

# 5. 檢查 Doctor
openclaw doctor
```
