# 三層擴充架構 — Extension、Skill、Plugin 的完整模型

> **摘要**：OpenClaw 的可擴充性建立在三層架構之上：**Extensions**（121 個，基礎設施層擴充）、**Skills**（53 個，高階任務描述）、以及 **External Plugins**（透過 ClawHub 市集或 npm 安裝的第三方擴充）。Extension 是程式碼模組，提供新的 Provider、Channel、Tool 等能力；Skill 是 Markdown 文件，描述如何使用工具完成任務；Plugin 則是統稱所有可載入模組的概念。三者透過 Plugin Loader 統一管理，具有完整的發現、載入、啟用、衝突解決機制。`enabledByDefault` 控制哪些擴充預設啟用，而 ClawHub 市集是分發第三方擴充的官方管道。

## 2. Extension vs. Skill vs. Plugin 的區別

### 2.1 Extension：程式碼模組

Extension 是 OpenClaw 的**基礎設施擴充**。它是 TypeScript 程式碼模組，打包在 `extensions/` 目錄下，透過 npm 分發。

**一個典型的 Extension（以 Discord 為例）**：

```json
// source-repo/extensions/discord/package.json:1-69（關鍵部分）
{
  "name": "@openclaw/discord",
  "version": "2026.4.12",
  "type": "module",
  "peerDependencies": {
    "openclaw": ">=2026.4.12"
  },
  "openclaw": {
    "extensions": ["./index.ts"],          // 程式碼入口
    "setupEntry": "./setup-entry.ts",       // 設定精靈入口
    "channel": {
      "id": "discord",                      // Channel ID
      "label": "Discord",                   // 顯示名稱
      "selectionLabel": "Discord (Bot API)",
      "markdownCapable": true,              // 支援 Markdown
      "configuredState": {
        "specifier": "./configured-state",
        "exportName": "hasDiscordConfiguredState"
      }
    },
    "install": {
      "npmSpec": "@openclaw/discord",       // npm 安裝規格
      "minHostVersion": ">=2026.4.10"       // 最低主機版本
    },
    "compat": {
      "pluginApi": ">=2026.4.12"            // Plugin API 相容範圍
    },
    "build": {
      "openclawVersion": "2026.4.12"        // 建構時的 OpenClaw 版本
    },
    "bundle": {
      "stageRuntimeDependencies": true      // 階段化 runtime 依賴
    },
    "release": {
      "publishToClawHub": true,             // 發布到 ClawHub
      "publishToNpm": true                  // 發布到 npm
    }
  }
}
```

> — `source-repo/extensions/discord/package.json:1-69`

**Extension 的特點**：

- 是**程式碼**（TypeScript），需要編譯和載入
- 提供底層能力：Provider、Channel、Tool、Hook、Memory
- 透過 `package.json` 的 `openclaw` 區塊宣告 metadata
- 有版本相容性管理（`compat.pluginApi`、`install.minHostVersion`）
- 可以提供設定精靈（`setupEntry`）

### 2.2 Skill：任務描述文件

Skill 是**高階任務描述**，不是程式碼。它是 Markdown 文件（`SKILL.md`），告訴 Agent 如何使用特定的工具完成任務。

**一個典型的 Skill（以 Discord 為例）**：

```markdown
<!-- source-repo/skills/discord/SKILL.md:1-42（簡化） -->
name: discord
description: "Discord ops via the message tool"
metadata:
  openclaw:
    emoji: "🎮"
    requires:
      config: ["channels.discord.token"]
allowed-tools: ["message"]

## 使用指南
- 一律使用 channel: "discord"
- 遵守 gating：channels.discord.actions.*
- 偏好明確的 ID：guildId, channelId, messageId, userId
- 多帳號：可選的 accountId

## 動作範例
- 發送訊息：{ "action": "send", "channel": "discord", "to": "channel:<id>", "message": "hello" }
```

> — `source-repo/skills/discord/SKILL.md:1-42`

**另一個 Skill（GitHub）**：

```markdown
<!-- source-repo/skills/github/SKILL.md:1-45（簡化） -->
name: github
description: "GitHub operations via `gh` CLI"
metadata:
  openclaw:
    emoji: "🐙"
    requires:
      bins: ["gh"]
    install:
      - method: brew
        command: "brew install gh"
      - method: apt
        command: "sudo apt install gh"

## 使用指南
- ✅ 使用：PR 狀態、CI 運行、issue 操作、API 查詢
- ❌ 不要：本地 git 操作（直接用 git）
```

> — `source-repo/skills/github/SKILL.md:1-45`

**Skill 的特點**：

- 是**文件**（Markdown），不是程式碼
- 描述任務如何完成，不提供新能力
- 有 metadata（emoji、requires、install）
- 指定允許的工具清單（`allowed-tools`）
- 被注入到 Agent 的 system prompt 中
- 不需要編譯或載入，只需要文字讀取

### 2.3 Plugin：統稱概念

在 OpenClaw 的內部實作中，「Plugin」是一個更廣泛的概念。任何可以被 Plugin Loader 載入的模組都是 Plugin。Extension 本質上就是一種 Plugin，只是 Extension 特指 `extensions/` 目錄下的打包模組。

```typescript
// source-repo/src/plugins/manifest.ts:148-189（簡化）
PluginManifest {
  id                           // 唯一 ID
  configSchema?                // JSON Schema 配置驗證
  enabledByDefault?            // 是否預設啟用
  legacyPluginIds?             // 舊版 ID 映射
  autoEnableWhenConfiguredProviders?  // 配置 provider 時自動啟用
  kind?                        // 類型：channel, provider, tool, service, memory...
  channels?                    // 擁有的 channel 清單
  providers?                   // 擁有的 provider 清單
  modelSupport?                // 模型家族支援（prefix, pattern）
  activation?                  // 啟用觸發條件
  setup?                       // 設定 metadata
  contracts?                   // 能力合約
}
```

> — `source-repo/src/plugins/manifest.ts:148-189`

### 2.4 三者的關係表

| 面向 | Extension | Skill | External Plugin |
|------|-----------|-------|----------------|
| **本質** | TypeScript 程式碼模組 | Markdown 任務文件 | TypeScript 程式碼模組 |
| **位置** | `extensions/` 目錄 | `skills/` 目錄 | npm、ClawHub、本地路徑 |
| **提供** | Provider、Channel、Tool、Hook | 任務描述、工具指引 | 同 Extension |
| **載入** | Plugin Loader（Jiti） | 文字讀取注入 prompt | Plugin Loader（Jiti） |
| **版本** | 隨 OpenClaw 發布 | 隨 OpenClaw 發布 | 獨立版本管理 |
| **開發** | 需要 Plugin SDK | 只需寫 Markdown | 需要 Plugin SDK |
| **數量** | 121 個 | 53 個 | 不限 |
| **分發** | 內建打包 | 內建 | ClawHub / npm |

## 3. Plugin 發現與載入機制

### 3.1 四個 Plugin 來源

Plugin 可以來自四個不同的來源（Origin），每個來源有不同的優先順序：

```typescript
// source-repo/src/plugins/manifest-registry.ts:60-65
PLUGIN_ORIGIN_RANK = {
  config:    0,  // 最高優先 — 配置檔明確指定
  workspace: 1,  // 工作空間本地
  global:    2,  // 全域安裝
  bundled:   3   // 最低優先 — 隨 OpenClaw 打包
}
```

> — `source-repo/src/plugins/manifest-registry.ts:60-65`

| 來源 | 優先順序 | 說明 | 典型路徑 |
|------|---------|------|---------|
| `config` | 0（最高） | 配置檔中明確指定的 plugin | `plugins.entries.<id>` |
| `workspace` | 1 | 工作空間本地的 plugin | `.openclaw/plugins/` |
| `global` | 2 | 全域安裝的 plugin | `~/.openclaw/plugins/` |
| `bundled` | 3（最低） | 隨 OpenClaw 打包的 extension | `extensions/` |

## 4. enabledByDefault 機制

### 4.1 定義

`enabledByDefault` 是 Plugin Manifest 中的一個布林值，決定 plugin 是否在沒有明確配置的情況下自動載入：


> — `source-repo/src/plugins/manifest-registry.ts:67-113`

### 4.2 語義

```
enabledByDefault = true  → Plugin 自動載入，除非使用者明確停用
enabledByDefault = false → Plugin 不載入，除非使用者明確啟用
```

## 6. 擴充的衝突處理

### 6.1 Origin 優先順序消除重複

當同一個 plugin 出現在多個 origin 時，系統保留最高優先的：

```typescript
// source-repo/src/plugins/manifest-registry.ts:60-65
// config(0) > workspace(1) > global(2) > bundled(3)
// 同一 plugin ID 的多個候選者 → 保留 rank 最低的（優先順序最高）
```

> — `source-repo/src/plugins/manifest-registry.ts:60-65`

例如：如果 `discord` plugin 同時存在於 config（使用者指定的路徑）和 bundled（extensions/discord），config 版本勝出。

### 6.2 工具名稱衝突

Plugin 的工具名稱不能與核心工具衝突：


### 6.3 HTTP 路由衝突

Plugin 註冊的 HTTP 路由有嚴格的衝突規則：

```typescript
// source-repo/src/plugins/http-registry.ts:12-96
registerPluginHttpRoute(params) {
  // 步驟 1：正規化路徑（有 fallback 路徑）
  // 步驟 2：檢查是否有重疊路由 + 不同 auth
  //   → 不同 auth 的重疊路由：拒絕，記錄，返回空 cleanup
  // 步驟 3：檢查精確路由衝突
  //   → 不是 replaceExisting：記錄衝突，返回空 cleanup
  //   → 不同 pluginId 嘗試替換：記錄拒絕，返回空 cleanup
  //   → 同一 pluginId + replaceExisting：允許替換
  // 步驟 4：註冊路由 entry + metadata
  // 步驟 5：返回 cleanup 函式
}
```

> — `source-repo/src/plugins/http-registry.ts:12-96`

**衝突規則總結**：

| 情境 | 行為 |
|------|------|
| 不同 auth 的路由重疊 | 拒絕新路由 |
| 同路徑、不要求替換 | 拒絕新路由（記錄衝突） |
| 同路徑、不同 plugin 要求替換 | 拒絕替換 |
| 同路徑、同 plugin、要求替換 | 允許替換 |

## 7. ClawHub 市集

### 7.1 ClawHub 是什麼？

ClawHub 是 OpenClaw 的**官方擴充市集**（marketplace），用於發布和發現第三方 Plugin 和 Skill。它的角色類似 npm 之於 Node.js，但專門為 OpenClaw 生態系統設計。

VISION.md 明確指出 Skill 應該優先發布到 ClawHub：

> "Skills: Publish to ClawHub first (clawhub.ai), not core by default."
>
> — `source-repo/VISION.md:82`

### 7.2 安裝來源

Plugin 可以透過四種來源安裝：

```typescript
// source-repo/src/config/types.installs.ts:1-18
PluginInstallRecord.source = "npm" | "archive" | "path" | "clawhub"

// ClawHub 專屬欄位：
clawhubUrl?: string
clawhubPackage?: string
clawhubFamily?: "code-plugin" | "bundle-plugin"
clawhubChannel?: "official" | "community" | "private"
```

> — `source-repo/src/config/types.installs.ts:1-18`

| 安裝來源 | 說明 | 用途 |
|---------|------|------|
| `npm` | npm 套件 | 標準 npm publish 的 plugin |
| `archive` | 壓縮檔 | 直接下載的 tarball |
| `path` | 本地路徑 | 開發中的 plugin |
| `clawhub` | ClawHub 市集 | 市集安裝 |

### 7.3 ClawHub 頻道

```
official    — 官方發布的 plugin
community   — 社群貢獻的 plugin
private     — 私有的 plugin
```

### 7.4 ClawHub 家族

```
code-plugin   — 程式碼 plugin（完整 Extension）
bundle-plugin — 打包 plugin（Skill bundle 等）
```
